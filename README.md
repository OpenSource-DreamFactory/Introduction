 
## 介绍
开源梦工厂旨在建立一个从Web3.0到开源世界的门户，激励开发者持续为开源项目贡献代码和时间。我们在Polkadot上开发了一个动态NFT协议，它可以创建了与开源项目和贡献者活动相关联的动态NFT，参与者将竞争开发最有前途的项目，利用动态NFT和DAO机制来支持开源创新。
The Open Source Dream Factory aims to establish a gateway from Web3.0 to the opensource world, incentivizing developers to continue contributing code and time to open source projects. We have developed a dynamic NFT protocol on Polkadot，It creates dynamic NFTs associated with open source projects and contributors' activities, Participants will compete to develop the most promising projects that can leverage the dynamic NFT and DAO mechanisms to support open source innovation.

## 目标
1. 鼓励和激励开源开发者对项目的贡献
2. 为开源项目注入持续的资金来源

4. 实现DAO治理，让社区参与决策

开源梦工厂、Gitcoin和GitHub Sponsor这三个项目的目标群体存在部分差异:
1. 开源梦工厂:
主要面向开源项目开发者社区和加密货币投资者。通过发行NFT和DAO等机制为开源项目提供资金支持,同时也满足投资者的游戏化需求。目标是激励开源开发和资助开源创新。
2. Gitcoin:
主要面向开源项目开发者社区。Gitcoin是一家非营利组织,通过打赏、赞助等方式直接为开源项目和开发者提供资金。目标是推动开源协作和创新。
3. GitHub Sponsor:
主要面向GitHub平台上的开源项目维护者。GitHub Sponsor允许用户资助他们常用的开源项目,资助的对象主要是GitHub上较活跃和有影响力的项目。目标是让依赖开源项目的开发者可以回馈维护者。
从上可以看出,这三个项目的目标群体主要集中在开源开发者社区,但侧重点有差异:
- 开源梦工厂主要面向群体是优质的开源项目（不管是web2，web3，还是其他），通过web3游戏化机制激发开源社区活力。
- Gitcoin直接面向开源项目及开发者提供赞助和资金支持。
- GitHub Sponsor主要面向GitHub平台上的开源项目维护者及其依赖者。
除了目标群体的差异,项目机制也有区别:
- 开源梦工厂采用NFT和DAO等区块链机制。
- Gitcoin采用直接捐赠和赞助。 
- GitHub Sponsor允许在GitHub平台上对项目进行订阅赞助。
所以,这三个项目虽然目标都是推动开源项目,但侧重点和机制实现存在一定差异。

## 特色和机制：动态NFT
对开源梦工厂项目来说,动态NFT是一个重要的特色和机制,但并不绝对必要。该项目的核心目标是:
- 激励开源开发者持续为开源项目贡献代码和时间。
- 为开源项目提供资金支持要达成这两个目标,项目可以采取其他机制,不一定依赖动态NFT。

例如:
1. 设立开源基金会,为开源项目和贡献者提供直接资助。该基金会的资金可以来自捐赠、赞助等。
2. 建立开源储蓄互助社区,社区成员定期缴费进入基金池,基金池的资金用来资助社区内的开源项目和活动。
3. 直接为开源项目和贡献者提供现金奖励和激励。可以设立评选机制,选出最活跃和贡献最大的开源项目和人员,给予奖励。
4. 提供其他非货币激励,如认证、排名、推广等。例如,可以对开源项目和贡献者进行排名和认证,提高知名度和影响力。
综上,动态NFT是开源梦工厂一个有创新意义的尝试,但项目的核心目标并不依赖与NFT。项目可以采取其他方式,如设立基金会、社区筹资、直接资助等手段来激励开源社区和为开源项目提供资金支持。如果去掉动态NFT机制,项目的技术实现难度会降低,但创新性也会减少。所以,动态NFT机制对该项目来说是一个加分项,但不是必要条件。项目团队需要权衡动态NFT带来的增值作用和技术难度,决定其在项目中的必要性。如果采用更简单直接的机制也可以达成项目目标,那么不使用动态NFT也是可行的选择。



## 在开源梦工厂项目中使用动态NFT来记录开源贡献的行为,有以下几个好处:
1. 提高项目的技术含金量和创新性。NFT和DAO是区块链技术的前沿概念,项目采用这些机制可以体现出较高的技术创新力,吸引更多技术社区的眼球。
2. 增强游戏化和社区参与感。为开源贡献发行NFT,可以gamefy游戏化整个开源生态系统,让用户产生收藏和社交的动力,进而主动为项目贡献。这可以有效调动社区活力。
3. 提高项目透明度。记录开源贡献的动态NFT可以在区块链上公开透明地标记每个贡献行为,这增加了项目的公信力,也便于社区监督。
4. 可在二级市场交易。项目发行的NFT可以在二级市场上进行交易,这为NFT持有者创造了价值,也可以成为项目的一个经济模型。
5. DAO治理的一致性工具。NFT可以成为DAO治理中的投票代币,NFT持有者拥有权重投票的权利。这可以推动DAO治理的进程。

但是,使用动态NFT也面临一定难度:
1. 开发技术难度较大。开发动态NFT和其发行与交易机制需要较深的区块链技术功底,这增加了项目的开发难度。
2. 运营成本较高。管理NFT生态系统和维护其经济模型需要投入大量人力资源,这会提高项目的运营成本。
3. 法律风险。NFT在不同地区的法律地位仍不明确,存在一定的监管和法律风险。

## 动态NFT解决了追溯与决策和分配问题

现在对于社区/DAO内部资产&资源，分配和决策，逻辑和事实上，分为两种形式。即事前和事后，而追溯属于事后。需要DAO/社区确认的贡献有三个方面，1、基本事实的记录；2、效果的确认；3、权益的确认；现在大多数的DAO/社区，对于以上三个方面是采用事前确认的方法，一般会有个治理委员会，并且搞一套规则，这个的好处很明显，就是便于记录基本事实，并且规避掉一些治理委员会认为不需要、不应该做的事情；但是贡献的事前确认的问题同样严重，其一、治理委员会客观上起到经济计划部门的作用，因为他们也只能分配一些积分或者币；其二、社区的发展水平也受限于治理委员会的能力和水平；其三、对于贡献确认的后两个方面，治理委员会客观上很难做出正确的决策和分配，因为一件事情只有做了，它的效果才会在一段时间内显现，所以效果确认很考较治理委员会的能力和水平，效果不确定的情况下实际上，权益更加难以确认、决策和分配；追溯，作为一种事后确认贡献的方式，它的问题很明显，其一、基本事实的记录会时常缺失；其二、分配的激励不及时，造成人员流失；其三、权益的分配需要根据事实和效果博弈（尤其是对于开拓性的事情）；但是好处也很明显，因为一个基本的事实是，一件事情只有做了，大家才会知道它的效果以及该如何分配权益，在效果和分配权益方面也更自然的形成，社区共识，不会依赖治理委员会，相对也不用赋予治理委员会过多的社区权力；
如何改进追溯这种方式？采用github式工作文档协作，采用动态NFT记录开源贡献的行为并且上链，为的是解决这个问题。

- 创建一个Chrome插件，方便用户查看和管理他们的NFT

### DAO基金池
- 创建DAO基金池，所有NFT销售所得ETH都进入基金池
- 基金池资金的使用需要DAO成员投票决定

### DAO治理
- 设计DAO治理规则，让NFT持有者能参与项目的治理

### 技术需求
- 选择适合的区块链技术和开发框架
- 开发动态NFT和DAO基金池的智能合约
- 开发Chrome插件
- 对所有智能合约进行安全审计
- 设计和开发数据接口，允许开发者上传贡献数据

### 运营需求
- 建立并管理社区，活跃社区氛围
- 设计并执行风险管理和监管策略
- 加强安全防御，预防黑客攻击和数据泄露
- 加强宣传推广，增加用户和投资者的参与度

### 系统架构
1. 前端：采用现代前端框架如React或Vue，进行开发者界面和投资者界面的构建。
2. 后端：主要为智能合约，运行在以太坊区块链上，管理NFT的发行、交易和升级，以及DAO基金池的管理。
3. 数据存储：利用区块链的特性，所有的交易数据和NFT状态都在区块链上公开透明地存储。
4. Chrome插件：基于JavaScript和Chrome插件API，用于展示用户的NFT和贡献等级。
